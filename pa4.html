Goal

In this tutorial you will partly complete the files src/glApi.cpp and src/glApi.hpp in order to be able to map textures on 3d objects.

Useful files

For this tutorial, you will find the following files useful:

  • src/glApi.hpp
  • src/glApi.cpp
  • examples/PA4Application.hpp
  • examples/PA4Application.cpp

As usual, begin by importing the latest base code updates into your repository by running 

git pull

Part 1: Textures

In this part you should start by looking at implementation in examples/PA4Application.cpp and get familiar with its implementation. However you will only modify the files named src/glApi.cpp and src/glApi.hpp.

Your only task here is to fill in the implementation of the Texture class. As always, have a look at the doxygen-like documentation in the src/glApi.hpp and then complete the implementation in src/glApi.cpp. You should take some particular care to the handling of the Texture::setData templated method.

Validation

To verify that your implementation works correctly, you can run: 

$ ./glitter pa4 1 # for part 1 (use 2 or 3 later on)
Rendering after the first part completion: notice the strong aliasing.

Par 2: Mipmaps

In this part, you should complete further the Texture::setData method to generate mipmaps when needed. Check that this features effectively mitigates the stong aliasing visible in part 1.

Part 3: OpenGL samplers

In this part, you should implement the Sampler class, in order to be able to set up texture sampling parameters. Notice in particular the effect of anisotropic filtering on the rendering of the checkerboard.
