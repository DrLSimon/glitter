Part 1: the git revision control system

The base code of glitter is provided as a git repository. git is the most widespread distributed version control system in use today—it is a good idea to familiarize yourself with this system if you've never used it before. As the semester progresses, we will push out a number of additions to the glitter framework. You can simply merge our new revisions into your repository to benefit from these improvements.

Recommended learning resources for git are: try git, git-the simple guide, the git immersion tutorial, and the extensive free Pro Git ebook. The remainder of this section describes the steps needed to clone and compile the glitter base code.

Cloning using the command line

The source code of glitter is available on github. To check out a copy of the source code, enter 

$ git clone --depth=1 --single-branch \
  https://github.com/DrLSimon/glitter

on the terminal. The two flags (--depth=1 and --single-branch) are only useful to save up disk space.

Updating the codebase

We'll likely publish extensions and bug-fixes to this repository as the semester progresses. To get them, navigate into the glitter directory and run the command 

$ git pull
If there were any concurrent modifications to the same file, git may ask you to resolve merge conflicts. We refer you to the learning resources for information about resolving conflicts in general.

Note: Since your repository will contain homework solutions it is (obviously) not OK to post it online (in particular, don't fork the glitter project on github or elsewhere). Private forks (i.e. without public access) are allowed, however.

Part 2: Setting up a C++ compiler and building the base code

Configuring on Linux / Mac OS X terminal

Begin by installing the CMake build system on your system. On Mac OS X, you will also need to install a reasonably up-to-date version of XCode along with the command line tools. On Linux, any reasonably recent version of GCC or Clang will work. Navigate to the glitter folder, create a build directory and start cmake, like so: 

$ cd path-to-glitter
$ mkdir build
$ cd build
$ cmake .. -DCMAKE_BUILD_TYPE=Release

Building on Linux / Mac OS X terminal

After the Makefiles are generated, simply run make to compile all dependencies and glitter itself. 
$ make -j 4
This can take quite a while; the above command compiles with four processors at the same time. Note that you will probably see many warning messages while the dependencies are compiled—you can ignore them.

Configuring and building on QtCreator

Start by opening the CMakeLists.txt file with QtCreator. Then a configuration menu will be presented.
Simply press Configure Project button.
Once the project is configured, you can build / run / debug the executables within QtCreator.
