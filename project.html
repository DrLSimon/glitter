What to submit

For this project, all your deliverable should be submitted by e-mail. Here is the list of delieverable that we expect from you:

  • A game design document (before the project sessions)
  • A url to your gitlab / github repository, which should include:
    1. The source code of your game. This code should be clean and follow OOP principles.
    2. A readme.md. This readme should at least mention the authors, license, dependencies, build steps and run instructions. It should also feature a video demonstrating typical gameplay.
    3. [Optional] a web page (for instance a github page).
    4. A 10 minute oral presentation of your project. This presentation will be held at the end of the last project session, preferably by a single member of the team (e.g the team leader).

Grading

Only code that you implemented will be graded. Code that was graded in previous assignments will not be considered for grading in the final project. Code that was not written during the course, by the students submitting the project, will not be considered for grading and must be properly documented.
