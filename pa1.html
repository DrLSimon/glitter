Goal

In this tutorial you will partly complete the files src/glApi.cpp and src/glApi.hpp in order to be able to render a 2d square made up of two triangles.

Useful files

For this tutorial, you will find the following files useful:

  • src/glApi.hpp
  • src/glApi.cpp
  • examples/PA1Application.hpp
  • examples/PA1Application.cpp
  • shaders/simple2d.v.glsl
  • shaders/simple2d.f.glsl

Part 1: Rendering without IBO

In this part you should start by looking at implementation in examples/PA1Application.cpp and get familiar with its implementation. However you will only modify the files named src/glApi.cpp and src/glApi.hpp.

For each of the following classes (class Buffer), look at the doxygen-like documentation in the src/glApi.hpp header file and complete the implementation of the class methods in src/glApi.cpp (or directly in the header for templates) so that their implementation matches with the documentation. The list of classes that you need to complete is here:

  • Buffer
  • VAO
  • Shader
  • Program
Note that some methods of the VAO and Program classes should be completed later on (in part 2 or in other assignments). The remaining classes will be implemented in later assignments.

Validation

To verify that your implementation works correctly, you can run: 

$ ./glitter pa1 1
Rendering after the first part completion: a yellow rectangle on white background.

Part 2: setting up a VAO including an IBO

As you may have noticed, the rendering in part 1 is based on an incomplete class VAO. Indeed, you have not completed yet the following methods

  • VAO::setVBO
  • VAO::encapsulateVBO
  • VAO::setIBO
  • VAO::draw
This has two consequences on the PA1Application class:
  1. some openGL calls are directly invoked in the code of PA1Application class
  2. no IBO was used, and therefore the draw call is glDrawArrays instead of glDrawElements and besides, some vertex positions are duplicated in the VBO

Implementation

To complete this part of the tutorial, you must complete the missing implementations of the VAO class. Note that for the VBO encapsulation, you can look at the openGL direct invocations in class PA1Application. However, the draw call shall be different since you will use the IBO encapsulated in the VAO.

Validation

To verify that your implementation works correctly, you can run: 

$ ./glitter pa1 2
The rendering should be identical to Part 1.
